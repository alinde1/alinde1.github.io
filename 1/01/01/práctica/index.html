Objetivo

El objetivo de esta práctica es medir el estado de ánimo de los trabajadores en una empresa a partir de los mensajes de Slack.

Procedimiento

He utilizado el API de Slack [2] para leer los mensajes de todos los canales públicos a los que tenía acceso dentro del Workspace de la empresa. Y con el texto de los mensajes he realizado un análisis de sentimientos tal como hemos hecho en clase.

El API es fácilmente accesible con python. Como no me ha funcionado el acceso a los objetos de python desde R, el transpaso de información entre ambos lo he realizado escribiendo y leyendo ficheros.

Aunque los canales son “públicos” en realidad sólo son accesibles si eres miembro del Workspace. Por lo tanto, no incluiré los datos por si hay alguna información sensible que se me haya pasado.

Descargar los mensajes

Descargamos los mensajes de todos los caneles obtenidos en el punto anterior y guardamos los mensajes de cada canal en un fichero (data/<id_canal>.json).

{python}
import json
import requests
from tinydb import TinyDB

with open('channels.json') as json_file:  
    channels = json.load(json_file)

endpoint = 'https://slack.com/api/channels.history'
api_key = '--aquí iría el api key--'

for ch in channels:
    latest = '0'

    file_name = 'data/' + ch['id'] + '.json'
    db = TinyDB(file_name)

    while True:
        url = endpoint + '?token=' + api_key + '&channel=' + ch['id'] + '&latest=' + latest
        response = requests.get(url)
        json_data = json.loads(response.text)
        for m in json_data['messages']:
            db.insert(m)
        if json_data['has_more']:
            latest = json_data['messages'][99]['ts']
        else:
            break

Los ficheros generados los leemos con R.

{r}
df.msg <- data.frame(channel=c(), user=c(), text=c(), ts=c())
df.reactions <- data.frame(ts=c(), reaction=c(), count=c())

for (ch in channels[,'id']) {
  ch.json <- jsonlite::fromJSON(paste0('data/', ch, '.json'))
  for (i in 1:length(ch.json$`_default`)) {
    ts <- ch.json$`_default`[[as.character(i)]]$ts
    ts <- as.integer(unlist(strsplit(ts,'[.]'))[1])
    day <- as.POSIXct(ts, origin="1970-01-01")
    user <- ch.json$`_default`[[as.character(i)]]$user
    message <- ch.json$`_default`[[as.character(i)]]$text
    if (is.null(ch.json$`_default`[[as.character(i)]]$subtype)) {
      df.msg <- rbind(df.msg,
                      data.frame(channel=ch,
                                 user,
                                 text=message,
                                 ts=day,
                                 stringsAsFactors = FALSE))
    }
    if (!is.null(ch.json$`_default`[[as.character(i)]]$reactions)) {
      reactions <- ch.json$`_default`[[as.character(i)]]$reactions
      df.reactions <- rbind(df.reactions,
                            data.frame(ts=day,
                                       reaction=reactions[,'name'],
                                       count=reactions[,'count']))
    }
  }
}

create_text<- function( tempchannel )
{
  print(nrow(tempchannel))
  # creating empty array
  textarrdf = c(" ")
  
  for(i in 1:nrow(tempchannel))

    slackdf = tempchannel[i, "TEXT"]

Análisis de sentimientos

Ahora realizaremos un análisis de sentimientos siguiendo los pasos que dimos en clase [4].

Procesamos el texto para limpiarlo un poco. Utilizamos la función clean.text aunque algunas de las opciones son para Twitter.

{r}
slack.text <- df.msg$text

clean.text <- function(text){
  new_text = text
  new_text = gsub("(RT|via)((?:\\b\\W*@\\w+)+)", "", new_text)
  new_text = gsub("@\\w+", "", new_text)
  
  new_text = gsub("[[:punct:]]", "", new_text)
  new_text = gsub("[[:digit:]]", "", new_text)

  new_text = gsub("http\\w+", "", new_text)
  new_text = gsub("[ \t]{2,}", " ", new_text)
  new_text = gsub("^\\s+|\\s+$", "", new_text)
  new_text = gsub("[\n]{1,}", " ", new_text)
  
  return(new_text)
}

slack.text <- clean.text(slack.text)
head(slack.text)

handle_error = function(x){
  y = NA
  
  try_error = tryCatch(tolower(x), error = function(e) e)
  
  if(!inherits(try_error, "error"))
    y = tolower(x)
  
  return(y)
}

procesado = clean.text(slack.text)
procesado = lapply(procesado, handle_error)
procesado = na.omit(procesado)

Parece que hay mucho enfado. Sin embargo, mirando algunos mensajes nos damos cuenta de que el problema es que los mensajes están en castellano y por lo tanto los clasificadores no funcionan muy bien.

Una opción para solventar este problema consiste en traducir el texto al inglés. Para ello hay un paquete de R: translateR. Pero es necesario tener una cuenta del cloud de Google o de Microsoft. En python tenemos el módulo googletrans [5] que traduce el texto utilizando el traductor de Google. Tiene el problema de que si hacemos las llamadas demasiado rápido nos corta la conexión. Por eso he incluido una pausa entre llamadas.

Guardamos el texto del objeto de R en un fichero y lo procesamos con python.

{r}
text <- unlist(procesado)
write(text, file="procesado.txt")
{python}
f = open('procesado.txt', "r")
lines = f.readlines()
f.close()

from  googletrans import Translator
import time
translator = Translator()

f = open('procesado_en.txt', 'w')
for l in lines:
    r = translator.translate(l, dest='en').text
    time.sleep(0.5) #print(r)
    f.write(r+'\n')
f.close()
